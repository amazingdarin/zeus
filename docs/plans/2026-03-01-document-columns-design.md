# 文档横向分栏容器块设计（Design）

## 背景
当前文档编辑器支持多种内置块（段落、列表、表格、图表、脑图等），但缺少横向分栏能力。用户希望在文档内插入可编辑的分栏容器，支持 `2/3/4/5` 列布局，并可在编辑中动态调整列数。

## 目标
1. 提供内置分栏容器块：`2列`、`3列`、`4列`、`5列`。
2. 分栏列内支持任意块类型（段落、列表、图片、表格、图表等）。
3. 支持在分栏容器工具栏中动态修改列数（2~5）。
4. 移动端保持横向分栏，通过横向滚动访问全部列。
5. 与现有块插入、slash 菜单、块拖拽体系保持一致。

## 非目标
1. 本期不支持分栏嵌套分栏。
2. 本期不支持列宽拖拽/比例自定义。
3. 本期不保证 Markdown 往返保留分栏语义（仅保证可降级导出）。

## 已确认约束
1. 列内内容范围：支持任意块，禁止分栏嵌套。
2. 窄屏行为：保持横向分栏，允许横向滚动（不自动折叠为单列）。
3. 列数调整：支持动态改列数（2/3/4/5）。

## 方案对比

### 方案 A：语义化节点 `columns + column`（推荐）
新增两个 Tiptap 节点：`columns`（容器）与 `column`（列），通过 schema 约束结构。

优点：
1. 结构语义清晰，可长期演进（后续可扩展列宽比例、拖拽调宽）。
2. 与现有自定义节点（chart/mindmap）一致，接入成本可控。
3. 能与块系统、插入命令、渲染样式协同。

缺点：
1. 需要新增节点扩展、node view、命令与菜单接入，改动面中等。

### 方案 B：复用 `table` 伪装分栏
用单行多列表格模拟分栏。

优点：
1. 开发快，复用现有 table 能力。

缺点：
1. 语义与交互不匹配（表格行为与分栏预期不同）。
2. 后续维护和扩展成本高。

### 方案 C：原子块 attrs 承载列内容
把分栏当 atom，自管内部子编辑器。

优点：
1. UI 控制力强。

缺点：
1. 与主文档流耦合差，历史/粘贴/导出复杂，不符合当前架构。

结论：采用 **方案 A**。

## 推荐架构

### 1) 节点模型
新增节点：
1. `columns`：`group: "block"`，`content: "column+"`，`attrs.count` 限制 `2|3|4|5`。
2. `column`：仅作为 `columns` 子节点使用，`content: "block+"`。

约束：
1. `column` 内不允许再出现 `columns`（命令层 + schema 层双重限制）。
2. 初始化时每列默认一个空段落，保证可编辑起点。

### 2) 插入与编辑命令
1. 新增命令：
   - `insertColumns({ count })`
   - `setColumnsCount({ pos, count })`
2. 列数变化策略：
   - 增列：末尾追加空列。
   - 减列：裁掉尾部列，并把被裁列内容迁移到最后保留列末尾，避免丢数据。

### 3) UI 与交互
1. 块菜单新增：`2列`、`3列`、`4列`、`5列`。
2. slash 支持：`/2col` `/3col` `/4col` `/5col`。
3. `columns` NodeView 顶部轻量工具条：
   - 当前列数
   - 快捷切换 2/3/4/5
   - `+列` / `-列`
4. 移动端样式：
   - 容器 `overflow-x: auto`
   - 按列数设置最小总宽度
   - 每列最小宽度保障可编辑性

### 4) 导出兼容策略
1. JSON 存储：完整保留 `columns/column` 结构。
2. Markdown 导出：降级为顺序块流（按列顺序展开），不保留视觉分栏语义。
3. Markdown 导入：不新增分栏解析，保持现有行为。

## 数据结构草案

```json
{
  "type": "columns",
  "attrs": { "count": 3, "id": "col-wrap-1" },
  "content": [
    {
      "type": "column",
      "attrs": { "id": "col-1" },
      "content": [{ "type": "paragraph" }]
    },
    {
      "type": "column",
      "attrs": { "id": "col-2" },
      "content": [{ "type": "paragraph" }]
    },
    {
      "type": "column",
      "attrs": { "id": "col-3" },
      "content": [{ "type": "paragraph" }]
    }
  ]
}
```

## 测试策略

### 单元测试
1. 分栏内容构造：`2/3/4/5` 默认结构正确。
2. 菜单接入：内置 block 列表包含四个分栏项。
3. 列数变更：增列、减列、减列内容迁移正确。
4. 禁止嵌套：列内插入分栏被拒绝。

### 回归测试
1. slash 插入分栏可用。
2. 列内编辑任意块不破坏文档结构。
3. 移动端视口下可横向滚动并编辑。

## 验收标准
1. 可插入 `2/3/4/5` 列分栏容器。
2. 每列可编辑任意块，且不允许分栏嵌套。
3. 可在容器工具条动态改列数，且不会丢失已有内容。
4. 移动端保持横向分栏并支持横向滚动。
5. 不影响现有块菜单、slash、拖拽和已有块编辑行为。

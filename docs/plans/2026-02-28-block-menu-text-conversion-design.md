# 块菜单文本基础块互相转化设计

## 1. 背景与目标

当前左侧块动作菜单仅支持“删除块”。为提升文档编辑效率，需要在该菜单中支持文本基础块的互相转化，避免用户通过多次删除/重建完成块类型调整。

本次目标：

1. 在块菜单中新增“转换为”能力。
2. 支持文本基础块之间互转。
3. 保留文本和行内格式，结构不兼容时安全降级。
4. 与现有拖拽、`+` 插入菜单、`/` 菜单协同，不引入交互冲突。

## 2. 已确认需求（决策）

### 2.1 转换范围

按已确认选项实现以下块类型互转：

- 段落
- 标题 1/2/3
- 可折叠标题 1/2/3
- 无序列表
- 有序列表
- 任务列表
- 引用
- 代码块

### 2.2 菜单交互

- 块菜单新增一级项“转换为”。
- 点击/悬停后展开二级菜单展示目标类型。
- 隐藏“当前同类型”目标。

### 2.3 内容保留策略

- 优先保留文本内容和行内 marks（粗体、斜体、链接等）。
- 结构不兼容时做安全降级（例如代码块转普通块时仅保留文本）。

### 2.4 多行到单块策略

当列表/代码块等多行结构转成单块（段落/标题/引用）时：

- 合并为一个目标块。
- 保留换行符。

## 3. 方案评估

### 方案 A（推荐）：块菜单子菜单 + 统一转换器

- 在现有块菜单加入“转换为”二级菜单。
- 在编辑器层实现统一 `convertCurrentBlock(targetType)`。
- 通过“类型识别 + 通用转换管线”处理互转。

优点：

- 交互清晰，符合现有菜单结构。
- 转换规则集中，便于测试和后续扩展。

缺点：

- 首次实现需要补齐类型识别与规范化转换逻辑。

### 方案 B：菜单项逐个调用既有命令

优点：实现快。
缺点：规则分散、边界不一致风险高。

### 方案 C：独立 Tiptap 扩展统一提供转换命令

优点：架构最整洁。
缺点：工程量过大，不适合当前增量需求。

## 4. 推荐设计

### 4.1 结构层

在 `DocEditor` 中新增转换相关能力，复用当前块定位能力：

- `currentBlockId`
- `getTopLevelBlocks(editor)`
- `findTopLevelBlockById(editor, blockId)`

新增纯函数：

1. `resolveCurrentBlockConvertType(node)`：把顶层节点映射到“基础文本块类型”。
2. `getConvertibleTargetTypes(currentType)`：给出目标列表并排除同类型。
3. `convertBlockNodeContent(sourceNode, targetType)`：生成目标节点内容（含安全降级）。

### 4.2 交互层

在现有 `doc-editor-block-action-menu` 中新增：

- 一级项：`转换为`
- 二级项：各目标块类型（不含当前类型）
- 现有项：`删除块`（保留，置于分割线下）

菜单状态：

- 打开块菜单：现有 `Cmd+/` 或 `Alt+/` 保持不变。
- 点击转换项执行后关闭菜单并更新 handle。
- `Esc` 关闭菜单/子菜单。

### 4.3 数据流

`handleConvertCurrentBlock(targetType)` 流程：

1. 基于 `currentBlockId` 定位源顶层块。
2. 解析源块类型（用于过滤同类型与转换策略）。
3. 生成目标节点 JSON（保留文本与 marks，必要时降级）。
4. 通过事务原子替换源块范围。
5. 调整光标到新块可编辑位置。
6. 关闭菜单并调用 `updateBlockHandlePosition()`。

## 5. 转换规则细则

### 5.1 标题系

- `heading-1/2/3 <-> collapsible-heading-1/2/3`
- 仅调整 `level` 与 `collapsible` 属性，文本与 marks 保留。

### 5.2 段落/引用/标题互转

- 保留文本与行内 marks。
- 多行文本在目标单块中保留换行。

### 5.3 列表互转

- 单块转列表：按换行拆分为多个 list item。
- 列表转单块：合并 list item 文本，使用换行连接。
- 任务列表转其他列表：去除任务状态，仅保留文本。

### 5.4 代码块互转

- 转代码块：按纯文本写入 code block（行内 marks 降级）。
- 代码块转单块：文本保留，换行保留。

## 6. 异常与降级策略

1. 当前块不存在：直接返回，不做修改。
2. 源块类型不在转换集合：不展示“转换为”。
3. 事务失败：捕获并中止，不破坏原文档。
4. 与删除逻辑并存：转换不删除文档根结构，删除逻辑继续保底一个空段落。

## 7. 测试设计

### 7.1 单元测试

新增覆盖：

- 类型识别函数：paragraph/heading/collapsible/list/blockquote/codeBlock。
- 目标过滤函数：隐藏当前同类型。
- 内容转换函数：
  - 标题与可折叠标题互转
  - 列表 -> 段落（换行合并）
  - 段落多行 -> 列表（按行拆项）
  - 段落 -> 代码块（marks 降级）

### 7.2 编辑器集成测试

- 打开块菜单 -> 转换为目标块 -> 断言文档 JSON。
- 验证 `Cmd+/` / `Alt+/` 仍可唤起菜单。
- 验证删除块能力不受影响。

### 7.3 UI 自动化回归（后续）

- Playwright 用例覆盖关键转换链路与菜单交互。

1. 抽离“类型识别 + 目标过滤 + 内容转换”纯函数。
2. 在动作菜单接入“转换为”二级菜单。
3. 增加 `handleConvertCurrentBlock` 并联调现有状态收口。
4. 补单元与集成测试。
5. 再跑一次 Playwright 回归。
